# CloneTracker v1.0.0 — Руководство оператора

## Что это

CloneTracker — программа для учёта клонов при высокопроизводительном скрининге мутантов Aspergillus niger. Заменяет бумажный журнал и Excel-таблицы. Работает оффлайн, данные хранятся локально.

---

## Запуск

Двойной клик по `CloneTracker.exe`. Установка не требуется. При первом запуске Windows может показать предупреждение SmartScreen — нажмите "Подробнее" -> "Выполнить в любом случае".

Данные автоматически сохраняются в файл `clonetracker-data.json` рядом с exe.

---

## Логика работы (пайплайн)

```
Чашка Петри (мутагенез NTG/UV/CRISPR)
  |
  v
1. ПИКИНГ -> Source plate (96-well, S01, S02...)
  |         Оператор отмечает лунки с колониями
  |
  v
2. ПАССАЖ -> Passage plate (96-well, P01, P02...)
  |         Копия source plate, ожидание споруляции
  |
  v
3. TRANSFER -> Culture plates (48-DWP, C01, C02...)
  |           96 клонов -> несколько 48-well планшетов
  |           Каждый клон в 2x или 3x повторах
  |
  v
4. СКРИНИНГ -> Импорт OD данных с ридера
  |           Heatmap + ручная правка значений
  |
  v
5. РАНКИНГ -> Автоматический расчёт mean, SD, ratio vs WT
             Отбор лучших клонов
```

## Интерфейс

### Шапка (всегда видна)

| Элемент | Описание |
|---------|----------|
| **CloneTracker** | Логотип, версия |
| **Undo/Redo** | Отменить / Повторить (также Ctrl+Z / Ctrl+Shift+Z) |
| **Backup** | Экспорт бэкапа всей базы в .json |
| **Import** | Импорт бэкапа из .json |
| **Тема** | Переключение светлая / тёмная тема |
| **Табы** | Эксперименты - Планшеты - Анализ - Пайплайн - Статистика |

### Табы

1. **Эксперименты** — список всех экспериментов, создание новых
2. **Планшеты** — работа с планшетами выбранного эксперимента
3. **Анализ** — импорт OD данных, интерактивный heatmap, редактирование значений
4. **Пайплайн** — визуальная схема эксперимента (какие планшеты связаны)
5. **Статистика** — поиск клонов, survival rate, OD гистограмма, ранкинг

## Пошаговая работа

### Шаг 1: Создать эксперимент

1. Таб **Эксперименты** -> кнопка **+ Новый**
2. Заполнить:
   - **ID** — уникальный код (NTG1, UV2, CRISPR3...)
   - **Тип** — метод мутагенеза (NTG, UV, CRISPR, EMS, ARTP, Other)
   - **Описание** — краткое описание
   - **Заметки** — kill rate, условия и т.п.
3. Нажать **Создать**

### Шаг 2: Создать source plate (пикинг)

1. Таб **Планшеты** -> кнопка **+ Планшет**
2. Выбрать тип **Source 96**
3. Имя присваивается автоматически (S01, S02...)

### Шаг 3: Отметить клоны

**Вариант А — мышью (drag-selection):**
1. Выбрать планшет S01
2. Зажать мышь и протянуть по лункам с колониями
3. Отпустить -> появятся кнопки действий:
   - **Клоны** — отметить как клоны (зелёные)
   - **WT** — wildtype контроль (жёлтые)
   - **BL** — blank (серые)
   - **Мёрт** — нет роста (красные)
   - **Очист** — очистить лунки
4. Клик по букве строки (A-H) = выделить всю строку
5. Клик по цифре столбца (1-12) = выделить весь столбец

**Вариант Б — импорт из Excel:**
1. Кнопка **xlsx** на планшете
2. Скачать шаблон (ссылка "Шаблон")
3. Заполнить в Excel: 1/+ = клон, WT, BL, X = dead, пусто = пусто
4. Отдельно указать WT и Blank лунки в полях ввода
5. **Прочитать** -> **Применить**

**Вариант В — импорт из CloneCounter:**
1. Кнопка **CloneCounter** на source plate
2. Выбрать JSON файл, экспортированный из приложения CloneCounter
3. Программа прочитает координаты колоний и покажет сводку: источник, имя чашки, количество колоний, количество лунок для пикинга
4. **Прочитать** -> проверить данные -> **Применить**

Файл CloneCounter должен содержать поле `colonies` — массив объектов с полем `well` (например, `"A1"`, `"B3"`). Дополнительно могут быть поля `source`, `plate`, `date`.

**Вариант Г — анализ фото:**
1. Кнопка **Фото** на планшете
2. Загрузить фото планшета сверху
3. Кликнуть по 4 углам (верх-лево -> верх-право -> низ-право -> низ-лево)
4. Программа определит заполненные лунки по яркости
5. Подстроить порог слайдером, кликнуть по ошибочным лункам
6. **Применить**

### Шаг 4: Пересев (Transfer)

#### 96->96 (пассаж)
1. Кнопка **Transfer ->** -> выбрать source -> тип "96->96 (пассаж)"
2. **Предпросмотр ->** — откроется dual-panel:
   - Слева: source plate (тяни мышью для выделения области)
   - Справа: будущий passage plate (копия)
   - Выделить область на source -> кнопка "Исключить" = убрать клоны из пересева
3. **Подтвердить** -> создаётся passage plate (P01)

#### 96->48-DWP (скрининг)
1. Кнопка **Transfer ->** -> выбрать source -> тип "96->48-DWP"
2. Выбрать количество **повторов**: 2x (23 кл./пл.) или 3x (15 кл./пл.)
3. Выбрать **раскладку повторностей**:
   - **По строкам ->** — повторы идут горизонтально (К1.r1, К1.r2, К1.r3 в одной строке)
   - **По столбцам v** — повторы идут вертикально (К1.r1, К1.r2, К1.r3 в одном столбце)
4. **Предпросмотр ->**:
   - Тяни мышью по source plate для выделения -> "Исключить" или "Вернуть"
   - Перетаскивай клоны в списке внизу для изменения порядка (drag-and-drop)
   - Стрелки **< >** для листания между 48-DWP планшетами
5. **Подтвердить** -> создаются culture plates (C01, C02...)

### Шаг 5: Работа с отдельными планшетами

На тулбаре каждого планшета доступны кнопки:
- **Копия** — создать дубликат планшета (копируются все лунки)
- **Корзина** — удалить планшет (с диалогом подтверждения)

Удаление эксперимента тоже требует подтверждения в модальном окне.

### Шаг 6: Редактирование клонов на 48-DWP

Если после пересева нужно поменять клоны:
1. Выбрать culture plate -> кнопка **Клоны**
2. Кликнуть по лунке -> выбрать статус (Клон/WT/Blank/Dead/Пусто)
3. Для клона — ввести Clone ID и Source Well
4. Изменённые лунки подсвечены жёлтой рамкой
5. **Сохранить**

### Шаг 7: Импорт OD данных (скрининг)

**Таб "Анализ":**
1. Выбрать culture plate (C01, C02...)
2. Кнопка **Загрузить .xlsx** -> загрузить файл с ридера
3. Или кнопка **Скачать шаблон .xlsx** -> заполнить вручную -> загрузить
4. Данные отобразятся как heatmap
5. **Клик по лунке** -> ввести/исправить OD значение вручную
6. Enter = сохранить, пусто + Enter = удалить, Escape = отмена

**Альтернативно** (быстрый способ):
- Кнопка **OD .xlsx** прямо на планшете на табе "Планшеты"
- Или кнопка **Импорт** в шапке -> вставить текст / загрузить xlsx

### Шаг 8: Анализ результатов

**Таб "Статистика":**
- **Метрики**: кол-во планшетов, клонов, survival rate, dead count
- **OD гистограмма**: распределение значений
- **Поиск**: ввести ID клона -> увидеть все данные по нему
- **Ранкинг**: таблица клонов, отсортированных по mean OD
  - Mean+-SD, ratio vs WT, визуальный бар
  - Кнопка **Все эксперименты** — сравнить клоны между раундами мутагенеза

### Шаг 9: Экспорт и печать

**На каждом планшете (тулбар):**
- **xlsx** — экспорт планшета в Excel (карта клонов + статусы + OD)
- **Этикетка** -> три варианта:
  - **Этикетка** — печать наклейки 57x40 мм (QR-код + мини-карта, ч/б)
  - **PNG** — сохранить наклейку как файл 300 DPI
  - **Карта A4** — полноразмерная печать с цветовой кодировкой

**На уровне эксперимента:**
- **Excel** — все планшеты + сводка по клонам в одном файле
- **.gwl** — worklist для робота Tecan EVO150

## Настройки Tecan EVO150

Кнопка **Tecan** на табе Планшеты открывает форму настроек робота.

**Позиции на деке** (имена rack'ов в EVOware):
- Source 96-well (по умолчанию: Source96)
- Dest 96-well для пассажа (Dest96)
- Dest 48-DWP (Dest48)
- Petri для пикинга (Petri)

**Объёмы (uL):**
- Aspirate — объём забора (по умолчанию 5)
- Dispense — объём дозирования (по умолчанию 5)
- Transfer — объём на реплику при 96->48 (по умолчанию 10)

**Смена наконечников:**
- Каждый раз (always) — замена после каждого дозирования
- Между клонами (between_clones) — замена при переходе к новому клону
- Без смены (never) — без замены наконечников

**Wash station:** включить/выключить промывку.

Настройки сохраняются в localStorage и применяются ко всем генерируемым .gwl файлам.

## Бэкап и безопасность данных

### Автосохранение
Данные автоматически сохраняются:
1. В **localStorage** браузера (Electron)
2. В файл **clonetracker-data.json** рядом с exe (через 1.5 сек после каждого изменения)

### Ручной бэкап
- Кнопка **Backup** в шапке -> сохраняет полную копию базы в .json
- **Ctrl+S** — быстрый экспорт бэкапа (то же самое)
- Рекомендуется: делать бэкап в конце рабочего дня

### Предупреждение при закрытии
Если в базе есть данные и вы закрываете окно, программа покажет предупреждение "Вы уверены?".

### Восстановление
- Кнопка **Import** в шапке -> загрузить .json бэкап
- Данные полностью заменяются (с поддержкой Ctrl+Z для отмены)

### Перенос на другой компьютер
1. Экспорт бэкапа на старом компе -> .json файл
2. Скопировать CloneTracker.exe + .json на новый комп
3. Импорт бэкапа на новом компе

## Обозначения лунок (цвет на экране / паттерн на этикетке)

| Экран | Этикетка (ч/б) | Статус |
|-------|----------------|--------|
| Зелёный | Залитый | Клон (picked) |
| Жёлтый | Штриховка | WT контроль |
| Серый | Точка | Blank |
| Красный | Крестик | Нет роста (dead) |
| Тёмный | Пустой | Пустая лунка |

## Формат ID клона

```
{ЭКСПЕРИМЕНТ}-{ПЛАНШЕТ}-{ЛУНКА}
```
Пример: `NTG1-S01-B7`

Присваивается при пикинге, **не меняется** при пересадках. Это главный идентификатор клона через все этапы.

## Tecan .gwl worklist

Генерируется кнопкой **.gwl** на табе Планшеты. Формат:
```
C;Transfer: S01 -> 48-DWP
A;Source96;1;10;;
D;Dest48;1;10;;
W;
```
Загрузить в EVOware Standard -> Scripts -> Run Worklist.

Доступно 4 режима генерации: пикинг, пассаж, трансфер 96->48, полный пайплайн. Параметры настраиваются через кнопку **Tecan**.

## Решение проблем

| Проблема | Решение |
|----------|---------|
| Программа не запускается | Проверь что Windows 10/11 64-bit. Попробуй запустить от имени администратора |
| SmartScreen блокирует | "Подробнее" -> "Выполнить в любом случае" |
| Данные пропали | Импортировать бэкап или скопировать clonetracker-data.json |
| Этикетка не печатается | Проверь настройки принтера: размер бумаги 57x40 мм, без полей |
| OD значения не импортируются | Проверь формат: 8 строк x 6 столбцов, числа через точку (0.123) |
| Файл CloneCounter не читается | Проверь что это JSON с полем "colonies" (массив объектов с полем "well") |
| .gwl не работает в EVOware | Проверь имена rack'ов в настройках Tecan — они должны совпадать с конфигурацией деки |
